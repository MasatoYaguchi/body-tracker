## 概要
これまでタブ切り替えのみで制御されていた画面遷移のURLルーティング化 (`react-router-dom`導入) と、ランキングページの公開アクセス対応を行いました。
これにより、未ログインユーザーでもランキングを閲覧できるようになり、その際は個人情報保護のためユーザー名を匿名化して表示するようにしました。

## 変更内容
### フロントエンド
- **ルーティング導入**: `react-router-dom` (v7) を導入し、`App.tsx` を `BrowserRouter` 構成に変更。
  - `/`: ダッシュボード（認証必須・ログインへリダイレクト）
  - `/ranking`: ランキング（公開・未ログイン可）
  - `/login`: ログイン画面（ゲストのみ）
- **UIコンポーネント改修**:
  - `UserHeader`: ナビゲーションを `Link` に変更し、未ログイン時の表示（ログインボタン）を追加。
  - `MainLayout`: 共通レイアウトコンポーネントとして切り出し。
- **API通信**: `authApi` に認証不要の `fetchPublic` メソッドを追加。

### バックエンド
- **API公開化**: `GET /api/ranking` の `authMiddleware` を必須から任意に変更。
- **匿名化ロジック**:
  - `Authorization` ヘッダーがない、または検証失敗時はゲストリクエストとして処理。
  - ゲスト閲覧時は、ランキングデータの表示名を "User A", "User B"... 形式に変換して返却。

## 影響範囲
- アプリケーション全体の画面遷移ロジック
- `/ranking` ページの表示仕様
- `/api/ranking` APIのレスポンス仕様

## 動作確認
- [ ] **ゲストアクセス確認**: ログアウト状態で `/ranking` にアクセスし、ランキングが表示され、名前が匿名化されていること。ログインボタンが表示されること。
- [ ] **認証アクセス確認**: ログイン状態で `/` (ダッシュボード) が表示され、`/ranking` では本来のユーザー名が表示されること。
- [ ] **ルーティング挙動**: 未ログインで `/` にアクセスすると `/login` へリダイレクトされること。

## 補足
- 匿名化のロジックは単純なアルファベット置換 (User A, B, ...) としています。
- ランキングAPIは誰でも閲覧可能になったため、将来的にアクセス制限が必要な場合は別途検討が必要です。
